Cryptographic hash

This is an interactive simulation of the MD2 hash algorithm. This hash was introduced by Ronald Rivest in 1989. It is no longer secure and has been replaced by others such as SHA-256. But MD2 is simple to understand as a basis for learning about hash algorithms.

Hashing consists of processing messages of arbitrary length to a unique hash value of fixed length such that any change in the message results in a change in the hash value. This result is valuable. You can verify that a message hasn’t been tampered with by running the message through the hash algorithm and ensuring that the hash value hasn’t changed from that supplied with the message. Even a tiny change in the message will result in a large and obvious change in the hash. In that case, you know that the message has been tampered with.

After you enter or edit a message, click the Hash button and see the result. Then copy the message and hash. Then make a trivial change in the message, hash, and see how the hash result has changed.
